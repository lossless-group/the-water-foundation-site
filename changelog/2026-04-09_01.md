## Astro v6 migration (Content Collections)
### Content config migration
- Added new Astro v6 content config file:
  - `src/content.config.ts`
- Existing legacy file still present:
  - `src/content/config.ts`

### Added loaders to content collections
Astro v6 requires loaders for collections.
- Updated `src/content/events/config.ts`:
  - Added `loader: glob({ pattern: '**/*.md', base: './src/content/events' })`
  - Imported `glob` from `astro/loaders`
- Updated `src/content/facts/schema.ts`:
  - Added `loader: glob({ pattern: '**/[!_]*.md', base: './src/content/facts' })`
  - The `[!_]*` exclusion prevents underscore-prefixed markdown (e.g. `_All-Facts.md`) from being treated as a facts entry and failing schema validation.

### Render API change
Astro v6 changed how collection entries render.
- Updated `src/components/facts/FactCard--w-Image.astro`:
  - Replaced `entry.render()` with `render(entry)`
  - Updated import to `import { render, type CollectionEntry } from 'astro:content'`

### Events pages compatibility (slug handling)
With v6 loaders, pages relying on `event.slug` could fail.
- Updated events pages to derive a slug from `event.id` when `event.slug` is absent.
  - Added helper `getEventSlug = (event: any) => (event.slug ?? event.id?.replace(/\.md$/, ''))` in several pages.
