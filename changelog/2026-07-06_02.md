# Dependency maintenance pass — Vitest 4.1.5→4.1.10, @types/node 25.6.2→26.1.0

## Why Care?

Regular maintenance pass to keep this site's dependencies current now that it's settled on Astro 7.0.6. `pnpm outdated` surfaced three packages behind: two patch-range bumps handled by `pnpm update`, and one major bump (`@types/node`) applied explicitly. No source changes were required — no breaking changes surfaced.

## What's New?

| Package | Before | After |
|---|---|---|
| `vitest` | `^4.1.5` | `^4.1.10` |
| `@vitest/ui` | `^4.1.5` | `^4.1.10` |
| `@types/node` | `^25.6.2` | `^26.1.0` |

**Build result:** `Complete!` — 80 pages built, sitemap generated, no errors, both before and after the `@types/node` major bump.

## Verification

- `pnpm update` applied the in-range patch bumps for `vitest` and `@vitest/ui` (also incidentally moved `@types/node` to `25.9.4` within its old `^25` range).
- `pnpm update @types/node@latest` applied the major bump to `26.1.0` (outside the prior `^25` range).
- `pnpm build` run after each step — both green, 80 pages built each time.
- `pnpm outdated` returned empty after the final update — nothing left behind.
- `pnpm exec astro check` reports 64 pre-existing TypeScript errors, all in `src/pages/strategies/[slug].astro` (`entry.data` typed as `unknown` — a content-collection typing gap) plus a couple of unused-import warnings elsewhere. These predate this change (no source files were touched) and are unrelated to the dependency bump, so left as-is per maintenance-pass scope.

## Notes

- Unsplash image loading errors during prerender (`403 Forbidden`) are the same pre-existing API key issue documented in `changelog/2026-07-06_01.md` — not a regression from this pass.
- Version bumped `1.0.2.0` → `1.0.3.0` (third digit, per this site's 4-part versioning convention) since real dependency versions changed.
